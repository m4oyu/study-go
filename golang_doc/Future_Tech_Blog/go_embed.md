# Go 1.16からリリースされたgo:embedとは
こちらの記事\
https://future-architect.github.io/articles/20210208/


## 特徴
- go embed はコメントアウト（のようなもの）を使用してファイルを埋め込むモノ
- 同じことは os や io/ioutil で行うこともできるが io/ioutil に関しては非推奨
- ファイルならなんでもバイナリとして格納できる
- コメントアウトとの間は空けない
- importの方法が単一ファイルと複数とで異なる。
- webAPIを開発時に大きなメリットとなる（なんで？）
- passを使って指定することもできる。しかし親ディレクトリをさかのぼることはできない
- 関数内では使えない


## 用途
- 広いスコープで扱いたい設定情報を含むファイルに対して
- ファイルの情報がビルドしたプログラムに組み込まれるため、実行バイナリのみで実行できる。（embedしたファイルを削除しても動作する）
- デプロイの運用フローに組み込むことで簡略化できる部分がありそう
- アセットファイルがいらなくなる！
